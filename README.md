Task Manager (Flask + PostgreSQL)

Опис проєкту
Цей проєкт є міні-застосунком Task Manager, реалізованим із використанням фреймворку Flask та системи керування базами даних PostgreSQL.

Застосунок призначений для керування завданнями (Tasks) та пунктами їх виконання (Steps), які логічно пов’язані між собою зв’язком one-to-many.

У проєкті реалізовано повний **CRUD-функціонал** для обох сутностей:
- створення
- перегляд списку
- перегляд одного об’єкта
- редагування
- видалення

Інтерфейс побудований на HTML-шаблонах із використанням Jinja2 та має стриманий дизайн із напівпрозорими блоками.

---

Функціональні можливості
- Додавання, перегляд, редагування та видалення завдань
- Додавання кроків (steps) до кожного завдання
- Перегляд кроків для конкретного завдання
- Автоматичне керування пов’язаними даними
- Обробка помилок та перевірка коректності введених даних

---

Структура бази даних

Таблиця `Tasks`
- `id` – первинний ключ
- `title` – назва завдання
- `date` – дата виконання
- `hard` – складність завдання

Таблиця `Step_tasks`
- `id` – первинний ключ
- `task_id` – зовнішній ключ на `Tasks.id`
- `title` – назва кроку
- `something_about` – опис
- `hard` – складність кроку

Зв’язок між таблицями: one-to-many  
(одне завдання може мати кілька кроків)

Технічні деталі реалізації

Flask
- Використано маршрути (`@app.route`) для реалізації CRUD-операцій
- Дані з форм обробляються через `request.form`
- Відображення даних здійснюється за допомогою Jinja2-шаблонів

PostgreSQL
- Робота з БД реалізована через бібліотеку `psycopg2`
- Параметри підключення до БД описані у функції `get_db()` у файлі `main.py`

---

Транзакції
У проєкті використано транзакції PostgreSQL для виконання кількох взаємопов’язаних операцій в одному логічному блоці.

Приклад:
- створення нового завдання у таблиці `Tasks`
- одночасне створення початкового кроку у таблиці `Step_tasks`

Обидві операції виконуються в межах однієї транзакції.  
У разі помилки всі зміни скасовуються за допомогою `rollback`.

---

Тригери
У базі даних реалізовано **тригер PostgreSQL**, який автоматично виконує допоміжну логіку (наприклад, підрахунок або оновлення агрегованих даних, таких як кількість завдань).

Це дозволяє:
- перенести частину бізнес-логіки на рівень БД
- зменшити навантаження на Flask-застосунок
- забезпечити цілісність даних

Запуск проєкту

1. Скопіювати проєкт у будь-яку директорію
2. Перейти в директорію з проєктом через термінал
3. Запустити сервер PostgreSQL
4. Вказати параметри підключення до БД у `main.py` (функція `get_db()`)
5. Ініціалізувати Flask-застосунок:
   ```powershell
   $env:FLASK_APP="main.py"
   flask init_db
